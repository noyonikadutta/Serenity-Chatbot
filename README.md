# Emotion-Aware Memory Graph Chatbot

An intelligent conversational system that combines emotion-aware reasoning with structured memory modeling and LLM-based response generation to enable context-aware, empathetic dialogue.

---

## 🚀 Overview

Most conversational AI systems rely on implicit context and lack emotional continuity across interactions. This project introduces a hybrid architecture that explicitly models emotional states and conversational context using classical data structures, enhanced with LLM-based response generation.

The system is designed to simulate therapist-style conversations with improved emotional consistency and contextual awareness.

---

## 🧠 Key Features

- Emotion detection from user input using rule-based and contextual analysis
- Multi-turn context retention using hash-based memory structures
- Emotion transition modeling using graph data structures
- Temporal emotion tracking using queue-based memory
- Integration with Gemini API for natural and dynamic response generation
- Web-based chat interface for real-time interaction

---

## 🏗️ System Architecture

The system consists of the following components:

- **Frontend:** Web-based chat interface (HTML, CSS, JavaScript)
- **Backend:** Flask-based server handling logic and routing
- **Emotion Detection Module:** Identifies user emotions from input text
- **Memory Module:** Stores and retrieves conversational context
- **Emotion Graph Engine:** Tracks emotional transitions over time
- **Response Generator:** Uses Gemini API for generating responses

---

---

## 🔄 Workflow

1. User inputs a message
2. Emotion detection module extracts emotional state
3. Context memory retrieves relevant past interactions
4. Emotion graph updates transition state
5. Gemini API generates context-aware response
6. Response is displayed to the user
